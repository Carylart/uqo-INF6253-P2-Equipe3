  CHOIX DU RAISONNEUR
  -------------------------------------------------------
  Le projet recommande HermiT ou Pellet (raisonneurs OWL DL).
  Ces outils necessitent Java et s'integrent principalement
  via Protege. Dans ce pipeline Python autonome, on utilise
  owlrl qui est 100% Python et ne necessite pas Java.

  owlrl / RDFS_Semantics est suffisant pour GO car :
    - GO est principalement une hierarchie (subClassOf) avec
      des restrictions someValuesFrom (couvertes par OWL-RL)
    - Les incoherences detectables dans GO sont toutes
      identifiables avec RDFS + regles OWL-RL de base
    - Le profil RDFS_Semantics evite la generation de millions
      de triplets inutiles (owl:sameAs, cardinalites, nominals)
      que RDFS_OWLRL_Semantics produirait sur cette ontologie

  Couverture RDFS_Semantics :
    rdfs11 : subClassOf transitif
    rdfs2/3 : propagation domain / range
    rdfs9   : propagation rdf:type via subClassOf
  Hors portee : OWL DL complet, cardinalites, nominals,
                owl:sameAs, owl:hasValue, owl:oneOf
